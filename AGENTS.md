# AGENTS.md

## Project Overview

This is a Java/Gradle project for clone-pair comparison tests.

Main source files are under:

- `src/main/java/equivalent`
- `src/main/java/inequivalent`
- `src/main/java/unverified`

Test files mirror those packages under `src/test/java`.

Clone-pair classes follow this naming pattern:

- `ClonePair<ID>.java`
- `ClonePair<ID>NoDifferenceTest.java`
- `ClonePair<ID>DifferenceFindingTest.java`

## Build And Test Commands

Use the Gradle wrapper.

- Compile tests: `./gradlew compileTestJava`
- Run regular tests: `./gradlew test`
- Run timeout-sensitive tests: `./gradlew timeoutTest`
- Run full verification: `./gradlew check`

Regular `test` excludes tests tagged with `timeout`; `timeoutTest` runs those separately.

## Clone-Pair Test Conventions

When adding or updating tests for clone pairs:

- Keep tests in the same package as the target clone pair.
- Use the existing support classes in that package when possible.
- Prefer `ClonePairGenericInvocationTestSupport` for generic input/output capture.
- Prefer `ClonePairByteArrayTestSupport` for byte-array boolean comparisons.
- A `DifferenceFindingTest` should use the same input for `method1` and `method2`.
- When the task is specifically about different return values, ensure both invocations return normally and assert that the returned values differ.
- If a `DifferenceFindingTest` is added for a clone pair, remove the matching `NoDifferenceTest` when requested.
- Do not delete unrelated `NoDifferenceTest` files.

Generated difference tests should follow the existing style:

```java
/**
 * Generated by Codex (GPT-5.5).
 */
```

## Editing Rules

- Keep generated test edits scoped to the relevant package and clone-pair ID.
- Do not refactor clone-pair implementation classes unless explicitly asked.
- Do not revert unrelated user changes.
- Avoid changing formatting in unrelated files.
- Use ASCII in newly generated files unless the test input explicitly requires Unicode.

## Verification Before Finishing

After adding or deleting tests, run at least:

```sh
./gradlew compileTestJava
```

When new `DifferenceFindingTest` files are added, also run the targeted tests for those new classes when practical.
